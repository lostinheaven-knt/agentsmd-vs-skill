# 会话路由

会话路由功能允许您**根据客户来源市场，将新会话自动分配给最合适的客服人员或团队。** 通过创建不同的分组，您可以轻松实现跨国团队的协同办公与多语言支持。

## 功能概览

通过分组功能，你可以：
- **精准分流**：为不同市场（如 US / EU / SA / MENA）创建独立客服组
- **专业匹配**：确保熟悉特定语言或业务的客服负责对应的市场
- **优先级管理**：当客户同时符合多条规则时，系统按优先级自动分配。

## 设置入口
1. 登录 Genstore 商户后台，点击 **销售渠道** -> **Inbox**。
2. 在 **客服工作台** 处，点击 **进入**。
3. 在左侧的菜单栏中，点击 **设置** 下的 **分组**，在此页面，您可：
   - [新增分组](#创建分组)
   - [管理分组配置](#管理分组)


## 创建分组
创建分组可实现将会话精准分配给特定客服。点击右上角 创建分组 按钮，按以下步骤操作：
1. **设置分组名称**：该分组设置一个易识别名称，如 `US Support`。
   > 建议：名称需唯一，建议包含市场或团队角色，长度不超过 30 个字符。
2. **配置服务区域**：点击 **选择市场**，勾选**一个或多个**市场，如：US、EU、SA、MENA 等。
   > 持续迭代中：未来我们将支持基于客户 IP 归属地、地理位置等更多维度的客群细分，敬请期待。
3. **配置客服人员**：点击 **选择员工**，勾选**一个或多个**客服人员。注意：
   - 只有被选中的客服，才可接收来自该分组的会话。
   - 您可以根据语言能力、专业技能或时区来分配客服。
4. **配置优先级**：设置 0-99 之间的整数，数值越高，优先级越高。
   - 注意：不同分组的优先级必须唯一。
   - 示例：若某客户同时命中 `US Support`（优先级 50）和 `Global Team`（优先级 10），系统将优先分配给 `US Support`。
5. **会话路由策略**：接收选定市场的客户消息，并根据客服优先级自动分配会话。
6. 设置完成后，点击 **保存** 完成当前分组的创建。

## 管理分组

分组创建后将以卡片形式展示在分组列表页中，您可以查看：
- **分组名称**：卡片左上角标识，如：`US Support` / `Arabic Team`
- **会话来源**：会话来源类型，当前仅支持按照市场进行路由。
- **员工列表**：该分组下的客服人员列表。
- **会话路由策略**：当前路由策略说明，如`接收选定市场客户消息`  

**操作提示**
- 点击 **编辑** 图标可修改规则。修改仅对新会话生效，已存在的会话不受影响。
- 点击 **删除** 图标可删除该分组。注意：如需 删除 分组，请确保该分组下无关联会话。

## 会话分配逻辑

当一个新客户会话创建时，系统将按照以下优先级进行“漏斗式”判断：
1. **规则匹配**：实时检索客户来源市场是否命中了自定义分组。
2. **优先级判定**：若同时命中多个分组，系统自动分配至优先级数值最高的分组。
3. **客服分配**：将会话推送至最终匹配分组内的员工工作台。
4. **兜底逻辑**：若未匹配到任何分组，会话将进入“默认分组”，确保响应不遗漏。

::: tip
为了防止因规则未覆盖导致客户无人接待，系统内置不可删除的默认分组：
- **工作原理**：当新会话无法命中任何自定义分组时，会自动路由至此。
- **人员范围**：默认包含所有具有 Inbox 处理权限的人员，确保全员都能关注到此类消息。
- **配置建议**：由于路由策略不可修改或删除。
:::

## 分配场景示例说明

为了帮助您更好地理解多条规则并存时的判定逻辑，请参考以下示例：
- **多重匹配场景**：一名来自美国的客户进线，同时命中了 `US Support` (90) 和 `Global Team` (10)。由于 90 > 10，会话将精准路由至 `US Support`。
- **无规则覆盖场景**：一名来自法国的客户进线，若未设置相关规则，会话将自动进入默认分组。

| 会话来源 | 命中分组 | 优先级 (Priority) | 最终路由结果 |
| :--- | :--- | :--- | :--- |
| **美国 (US)** | `US Support` | **90 (最高)** | `US Support` |
| | `Global Team` | 10 | |
| **法国 (FR)** | (无匹配) | - | **默认分组** |

## 常见问题（FAQ）

### Q1：可以基于 IP 或语言路由吗？
> A：当前版本仅支持基于市场区域进行路由，后续版本将支持位置、IP、标签、意图等规则，敬请期待。

### Q2：一个客服可以属于多个分组吗？
> A：可以。系统会根据命中分组和优先级决定路由归属。

### Q3：如何通过优先级来优化分配逻辑？
> A：优先级是系统判定分配逻辑的核心依据。通过设置不同的数值，您可以确保会话总是流向最精准的分组。
  >  - **高优先级（大数值）**：用于特定、高价值或语言针对性强的分组（如 VIP 服务、多语言专业组）。
  > - **中低优先级（小数值）**：用于宽泛、通用型的区域分组（如 Global Team）。
  > - **阶梯建议**：推荐采用阶梯式数值（如 90 / 70 / 50 / 10），以便后续随时在不同层级间插入新的细分规则，让流量分配更灵活。

### Q4：是否支持手动转移会话？
> A：支持。即使系统自动分配后，客服仍可在 Inbox 中手动将会话转移给其他更合适的同事或分组。

:::
