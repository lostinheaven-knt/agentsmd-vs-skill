# 订单履约

当买家在店铺下单后，系统会根据商户的 **订单履约设置**（手动或自动），决定是否由系统自动向供应商下单。
此流程确保产品从供应商端顺利履约、发货，并将物流信息实时同步至店铺。

## 功能价值

* 支持自动或手动履约，满足不同运营需求
* 系统自动同步供应商发货状态与物流追踪号
* 减少人工操作，提高订单处理效率

## 订单履约模式

Genstore Dropshipping 提供两种履约方式：**手动发货模式** 与 **自动发货模式**。


### 手动发货模式

在手动履约模式下，商户需要手动点击 **发货** 按钮后，系统才会向供应商提交订单请求。
适用于订单量较少、希望手动核对发货信息的商家。

**操作步骤**

1. 买家下单并付款后，订单状态显示为 **待发货**。
2. 商户进入 **商店** -> **订单**，找到代发货的 Dropshipping 订单。
3. 点击 **Genstore 代发货** 按钮，随即点击 **订单发货**。系统会自动向供应商提交订单。
4. 供应商接单后开始处理发货，并生成物流追踪号。
5. 系统将物流信息同步至店铺，订单状态更新为 **已发货**。

> **提示：**
> 若订单包含多个供应商的产品，系统会自动拆分为多笔订单并分别履约。

::: tip
您可以通过设置物流模板来管理运费计算规则，请参考[配置物流运费方案](./operate-dropshipping-shipping-profiles.md)了解如何配置不同地区、不同重量或不同配送方式的运费。
:::

### 自动发货模式

在自动履约模式下，系统会在买家完成付款后自动向供应商提交订单，整个过程无需人工干预。
该模式适用于订单量较大或追求高效率运营的商家。

**操作步骤**

1. 买家下单并完成付款。
2. 系统自动向供应商提交订单。
3. 供应商发货后，物流追踪信息会自动同步至店铺。
4. 订单状态自动更新为 **已发货**。

> **建议：**
> 若您已与稳定供应商合作，建议启用 **自动发货模式**，可显著提升履约效率。


## 物流信息同步

* 当供应商完成发货后，系统会自动获取物流单号与承运商信息。
* 物流信息将实时同步至商户后台与买家订单页面。
* 买家可在 **订单详情** 中查看配送状态。
