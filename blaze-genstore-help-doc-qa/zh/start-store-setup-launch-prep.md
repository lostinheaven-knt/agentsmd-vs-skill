# 完成店铺合规与上线准备

在完成基础搭建后，您需要完善商店的政策页面、物流与税费、支付与结账流程，并通过预览与测试订单，确保商店各项设置就绪，顺利上线运营。

## 设置公共信息页

创建和完善商店的基础政策页面，保障客户权益，符合平台合规要求。

- **创建政策内容**：可复用 Genstore 模版或自定义填写隐私政策、退换货政策、配送政策和服务条款等信息。具体操作，可参考 [创建政策页面](./operate-store-design-themes-edit-guide-policy#创建政策页面-任选以下方式)
- **添加导航菜单**：在 **商店 -> 在线商店 -> 菜单** 中，将政策页面添加到底部菜单，方便用户访问。具体操作，可参考 [将政策页面添加到底部菜单](./operate-store-design-themes-edit-guide-policy#将政策页面添加到底部菜单)
- **在店铺前台展示店铺政策**：通过 **装修编辑器** 关联底部菜单，使政策信息在店铺页面底部展示给买家。具体操作，可参考 [在店铺前台显示政策菜单](./operate-store-design-themes-edit-guide-policy#在店铺前台显示政策菜单)

## 配置物流与税费

设置发货和税费规则，确保商店运营符合客户期望与当地法规要求。

- **物流设置**：
  - 路径：后台 **设置 -> 发货配送**
  - 配置发货地址与仓库。确保商店地址填写准确。如涉及多个发货地点，请分别添加至 **地点** 管理。具体操作，可参考 [地点管理](./operate-fulfillment-location.md)
  - 设置配送区域与运费规则。具体操作，可参考 [设置运费](./operate-fulfillment-shipping-profile-shipping-rate.md)
  - 设定默认包裹尺寸，提升物流效率。具体操作，可参考 [管理包裹](./operate-fulfillment-manage-package.md)
  - （可选）承运商授权：通过授权 Genstore 物流合作伙伴，您可在 Genstore 后台购买面单，集中管理物流履约。具体操作，可参考 [Genstore 物流服务](./operate-fulfillment-genstore-shipping.md)。
- **税费设置**：
  - 路径：后台 **设置 -> 税务**

> **提示**：合理配置物流与税费，有助于提升客户购物体验并确保商店合规运营。

## 配置支付与结账流程

优化支付流程和结账体验，确保客户顺利完成付款。

- **启用支付服务**：
  - 路径：后台 **设置 -> 支付**
  - 激活 [Genstore Payments](./operate-genstore-payments.md) 或受支持的 [第三方支付服务提供商](./operate-payments-third-party.md)。
    **注意**：推荐开通 Genstore Payments，可免除额外第三方交易手续费，并在后台集中管理收入款项。
- **配置结账信息**：
  - 进入路径：后台 **设置 -> [结账页](./settings-checkout.md)**
  - 设置需要收集的客户信息（邮箱、电话、地址）。
> **提示**：顺畅的结账体验能有效减少购物车放弃率，提升订单转化率。

## 预览与调试店铺

在商店上线前，全面检查各页面内容与功能，确保整体体验流畅无误。

- **页面检查**：
  - 核查首页、产品页、系列页、购物车页、结账页、博客页内容完整、展示正确。
- **导航与链接检查**：
  - 确认所有菜单链接有效，无死链或跳转错误。
- **响应式适配测试**：
  - 检查桌面端、平板端与手机端显示效果是否正常。
- **SEO 元数据核对**：
  - 确认各页面标题、描述、URL 结构完整友好，便于搜索引擎索引。

> **提示**：建议从客户视角完整浏览一遍店铺，模拟真实购物体验，发现并及时修正潜在问题。

## 创建测试订单

通过模拟下单，验证支付、发货、退款等关键流程是否正常。

- **测试成功交易**：下单并完成支付，确保支付流程正常。
- **测试失败交易**：模拟支付失败场景，验证系统提示准确。
- **测试退款与取消订单**：在后台操作退款或取消订单，确认功能正常。
- **测试发货与部分发货**：创建发货单并发送物流通知，核查流程完整。

> **提示**：测试过程中，同时核对系统发送的通知邮件（如订单确认、发货通知等）是否准确无误。

## 正式上线商店

完成全部检查后，解除商店密码保护，正式对外开放。

- **解除密码保护**：
  - 进入路径：后台 **商店 -> 在线商店 -> 偏好设置**
  - 移除密码保护，开放商店访问权限。具体步骤，可参考 [关闭密码保护](./operate-store-design-preference#关闭密码保护)
- **邀请客户重设账户密码**（如迁移客户数据）：
  - 发送账户激活或密码重置邮件，帮助客户顺利登录新店铺。
- **添加其他销售渠道（可选）**：
  - 连接其他电商平台或社交媒体渠道，拓展曝光。具体步骤，可参考：
    - [Google & YouTube 销售渠道](./expand-sales-channels-google.md)
    - [Facebook & Instagram 销售渠道](./expand-sales-channels-facebook.md)
    - [TikTok 销售渠道](./expand-sales-channels-tiktok.md)
- **启动营销活动**：
  - 准备首轮营销推广计划，吸引流量并促进首单转化。具体步骤，可参考：
    - [邮件营销](./expand-marketing-email.md)
    - [短信营销](./expand-marketing-sms.md)
    - [社交媒体营销](./expand-marketing-social-media.md)
    - [自动化营销](./expand-marketing-automated-email-overview.md)

> **提示**：上线初期建议实时监控流量、支付与订单数据，及时响应客户反馈，保障顺利起步。同时，建议启用 [Genstore Inbox](./expand-inbox.md)，实时接收客户留言与订单消息，提升响应速度与客户体验。

## 小结

通过完成以上设置，您的 Genstore 商店将满足基础合规要求，具备良好的客户体验与运营条件，准备正式上线。

接下来，您可以专注于营销推广与客户运营，开启品牌增长之路！
