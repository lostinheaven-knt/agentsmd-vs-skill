# Shopyy 店铺迁移

本指南介绍如何使用 **Genstore 迁移助手** 将您的 Shopyy 店铺相关数据（包括店铺信息、产品 ）迁移至 Genstore。

## 开始迁移

::: tip

* 店铺数据量较大时，迁移可能耗时较长。
* 迁移过程中请勿修改 API 权限，否则可能导致迁移失败。
* 尽量避免在迁移过程中对 Shopyy 店铺进行大规模数据修改，以防数据不一致。
* [安装 Genstore 迁移助手](./start-migrate-guide#安装-genstore-迁移助手)。

:::

### 步骤一：获取 Shopyy API 访问权限

由于迁移过程中需要通过 API 读取并同步店铺数据，您需要在开始迁移前先在 Shopyy 中开发应用并获取 API 访问权限，具体步骤如下

1. 登录 Shopyy 后台，访问 **店铺信息** -> **活跃店铺**。
2. 找到待迁移的店铺，点击 **进入后台**。
3. 进入 **店铺设置** -> **系统设置**。
4.  点击 **开发者**，进入开发者模块。
   >  **注意**：系统已默认安装开发者应用，权限配置较高。出于数据安全考虑，建议您新建应用。
5. 点击 **新建应用**，在 **应用名称** 处填写应用名称。
6. 系统已默认勾选所有 **API 只读权限项**，点击 **保存**。
7. 点击 Token 后的复制图标，获取后台 API 访问令牌。

### 步骤二：打开 Genstore 迁移助手

1. 登录 Genstore 商户后台，导航至 ​**应用**​。
2. 在 **已安装** 页签，找到 ​**Migration Tool**​。
3. 点击 ​**打开应用**​，选择 **Shopyy** 作为迁移来源，点击 ​**开始搬迁**​。

### 步骤三：店铺授权

1. 输入访问令牌，即 ​**后台 API 访问令牌**​。
2. 完成设置后，点击 ​**下一步**​。

### 步骤四：选择迁移模块

Genstore 支持将 Shopyy 的以下模块迁移至 Genstore：

| 模块       | 是否必选
| ------------ | ---------- |
| 店铺       | 可选  |
| 产品       | 可选   |

::: tip

* 在迁移产品时，您可以选择 **覆盖更新** 方式： 系统将根据已有的 handle 进行匹配，对已有内容进行数据覆盖更新，而非创建重复项。

:::



### 步骤五：自定义迁移设置

您可以根据筛选条件选择特定的数据进行迁移：

* 产品：按状态筛选（上架、下架）。

### 步骤六：开始迁移

1. 确认所有设置后，点击 ​**开始迁移**​。
2. 系统将开始处理迁移，您可在应用界面查看迁移进度。迁移时间取决于数据量大小，请耐心等待。
3. 迁移完成后，可进入 Genstore 后台进一步完善店铺设置，推荐阅读 [店铺快速上线 Checklist](./start-quick-launch-checklist.md)

## 查看迁移记录

完成店铺迁移后，您可在迁移助手应用内查看所有迁移历史记录，包括：

* **迁移时间**
* **迁移平台**​（Shopyy）
* **店铺域名**
* **迁移模块**
* **迁移数据结果**
* **查看异常详情（并支持导出异常数据，便于后续核对与排查）**
