# 克隆 Genstore 店铺
本指南介绍如何使用 Genstore 迁移助手 (Migration Tool)，将源 Genstore 店铺的数据（包括店铺、产品、主题装修等）克隆到目标 Genstore 店铺。

## 前置准备
在开始之前，请确保目标店铺（接收数据的店铺）已安装 [迁移助手](./start-migrate-guide#安装-genstore-迁移助手) 应用。

## 开始店铺克隆

### 步骤一：获取源店铺的 API 访问权限

在迁移过程中需要通过 API 同步数据，请先在源店铺（提供数据的店铺）中创建应用并获取访问令牌：

1. 登录源店铺的 Genstore 后台，进入 **Apps**。
2. 点击右上角的 **开发应用** -> **创建应用**。
3. 在弹窗中填写 **应用名称**，选择 **应用开发人员**（店主或有应用开发权限的员工），点击 **创建应用**。
4. 在 **概览** 页签，点击 **配置 API 访问范围** -> **编辑**，勾选所有以 `read_` 开头的权限项，完成操作后点击 **保存**。
5. 返回 **概览** 页签，点击 **安装应用**。
6. 在确认弹窗中，点击 **安装**。
7. 系统将跳转至 **API 凭据** 页签，在 **API Access token** 部分，点击 **显示一次令牌** 获取 **后台 API 访问令牌**。注意：当前访问令牌仅可查看一次，请妥善保存。

### 步骤二：切换至目标 Genstore 店铺
由于数据将被迁移到目标店铺，您需要切换到目标店铺进行后续操作：
1. 在 Genstore 操作后台左下角的账户区，点击账户名称后的 "..."，选择 **切换商店**。
2. 在弹出的店铺列表中，选择目标店铺。
3. 导航至 **应用** -> **已安装**，找到 **Migration Tool**。
4. 打开应用，选择 Genstore 作为迁移来源，点击 **开始迁移**。

### 步骤三：源店铺授权
1. 输入源店铺的 Genstore 店铺域名。
2. 输入访问令牌，即 **后台 API 访问令牌**。
3. 完成设置后，点击 **下一步**。

### 步骤四：选择克隆模块
选择需要从源店铺克隆的数据模块到目标店铺：
| 模块 | 是否必选 | 依赖模块 |
| --- | --- | --- |
| 店铺 | 必选 | - |
| 产品 | 可选 | 店铺 |
| 自定义页面 | 可选 | 店铺 |
| 博客 | 可选 | 店铺 |
| URL重定向 | 可选 | 店铺 |
| 菜单导航 | 可选 | 店铺 |
| 主题模版 | 可选 | 店铺 |

### 步骤五：开始克隆
1. 确认所有设置后开始店铺克隆。
2. 系统将开始处理迁移，您可在应用界面查看迁移进度。迁移时间取决于数据量大小，请耐心等待。
3. 克隆店铺完成后检查所有数据完整性。

## 查看迁移记录

完成店铺迁移后，您可在迁移助手应用内查看所有迁移历史记录，包括：

- **迁移时间**
- **迁移平台**（Genstore）
- **店铺域名**
- **迁移模块**
- **迁移数据结果**
- **查看异常详情**（并支持导出异常数据，便于后续核对与排查）
