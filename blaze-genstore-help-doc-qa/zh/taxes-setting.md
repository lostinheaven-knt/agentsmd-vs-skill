# 税务设置

在 Genstore 开启商业经营时，合理的税务配置是确保合规经营与利润保护的关键。本篇内容将为您介绍如何在 Genstore 中管理收税区域，并根据业务需求选择最适合的计税方式。

## 收税区域

收税区域与您的发货配送区域相关，系统默认从您的 [发货与配送](./operate-fulfillment-shipping-profile-configuration.md) 模版中同步已配置的国家或地区。

- **新增区域**：若税率设置列表中缺少目标国家或地区，请前往 **设置** -> **发货配送**，在 **物流配送** 添加该区域后，保存并返回税务页面进行设置。
- **停止收税**：若您暂时不希望对某国家或地区收税，可关闭该区域的 **收税开关**。
- **移除区域**：仅当某区域从发货区域中删除后，方可在税务列表中点击 **操作** -> **移除** 来移除该区域。关于如何移除发货区域中的地点，请参考 [物流配送](./operate-fulfillment-shipping-profile-configuration#配置发货点)。
- 对于已不在发货区域内的国家或地区，系统会提示：“该国家不在您的物流配送区域内”。您可选择保留或移除该区域。

## 选择计税模式

Genstore 支持针对不同的国家或地区分别灵活配置计税服务。您可以根据目标市场的税制复杂度，在以下三种模式中进行切换：
- [Genstore Tax](./taxes-setting-genstore-tax.md)
- [手动税务](./taxes-setting-manual-tax.md)
- [Avalara Tax](./taxes-setting-avalara-tax.md)

::: tip
切换不同的税务服务时，您原有的历史税务数据不会丢失。
:::

## 其他设置

### 设置含税或不含税价

在完成税务方案配置后，您还需要设置价格是否包含税费：

1. **Genstore** -> **设置** -> **税务**。
2. 在 **税务展示方式** 中，勾选 **产品价格含税** 或 **产品价格不含税**。

::: tip

若您配置了运费税率，则客户订单结算时运费金额默认不含税。运费税将与产品税合并后统一展示。

:::

## 计算税费依据

- 地址依据：目前 Genstore 手动与自动税务均采用客户下单收货地址进行计算税率。对于虚拟产品和自提产品默认采用客户账单地址进行计算税率。暂不支持商户自行设置。
- 产品金额依据：目前 Genstore 根据产品的折扣价来计算税费（`产品折扣价 = 产品价格 - 产品折扣 - 订单折扣`），订单税费等于所有产品税费累加。 注意：通过礼品卡支付的部分不包含在减去的折扣金额内。

### 税费计算舍入

目前 Genstore 使用的是产品级别的税收舍入原则。即每个产品的税费单独计算（**计算出的税费会被四舍五入到小数点后两位**），最终将所有产品的税费相加得出总税费。

注：目前四舍五入的小数点位数，默认从 **设置** -> **市场**，点击 **偏好设置** -> **价格舍入** 获取，若需要调整请至此处设置。

### 不同类型订单的税务计算规则说明

在订单处理过程中，平台将根据订单类型采用不同的税务计算逻辑，具体如下：

#### 涉及虚拟产品订单

- 优先以 **账单地址（Billing Address）** 作为税务计算依据。

#### 配送方式为本店自提的订单

- 优先以 **账单地址（Billing Address）** 作为税务计算依据。
- 若账单地址为空，则以 **自提地址（Pickup Address）** 作为税务参考地址。

#### 配送方式为商家配送的订单

- 默认以每个产品的 **发货地址（仓库地址）** 与 **收货地址（收货人地址）** 组合作为税务计算依据。
- 若产品通过第三方应用发货，且发货详细地址在 Genstore 为空时，系统将默认使用账单地址 (Billing Address) 作为税务地址参考。

#### Dropshipping 订单
- Genstore Dropshipping （Marketplace 来源）：产品订单以产品发货仓所在国家地址（系统提供）作为发货地址进行计税依据。
- Genstore Print on Demand（POD 产品订单）：由于无法获取供应商发货地址，遂使用店铺账单地址作为发货地址进行计税依据。
- 其他 Dropshipping 产品订单（例如：Dsers 等 Dropshipping应用）：同样无法获取供应商发货地址，遂使用店铺账单地址作为发货地址进行计税依据。
