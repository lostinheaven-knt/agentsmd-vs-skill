# Shopify 店铺迁移

本指南介绍如何使用 **Genstore 迁移助手** 将您的 Shopify 店铺数据（包括客户、产品、订单等）迁移至 Genstore。

## 前置准备

迁移前，请确保完成以下准备工作，以确保数据完整性和迁移顺利进行：

- **报备支付渠道**：如 Shopify 店铺已启用支付功能，请提前通知支付渠道，以避免因店铺变更导致支付受限或风险。
- **确认店铺状态**：确保 Shopify 店铺正常运营，无冻结、关闭或其他异常情况。
- **检查产品库存和订单状态**：建议仔细检查 Shopify 店铺的产品库存和订单状态，确保数据准确。
- **备份 Shopify 店铺数据**：强烈建议在迁移前备份 Shopify 店铺数据，以防意外情况。
- **清理无用数据**：可清理无用的草稿订单、已取消订单等，以减少迁移数据量，提高效率。
- [已安装 Genstore 迁移助手](./start-migrate-guide#安装-genstore-迁移助手)。

## 开始迁移

:::tip

- 店铺数据量较大时，迁移可能耗时较长。
- 迁移过程中请勿修改 API 权限，否则可能导致迁移失败。
- 尽量避免在迁移过程中对 Shopify 店铺进行大规模数据修改，以防数据不一致。
:::

### 步骤一：获取 Shopify API 访问权限

由于迁移过程中需要通过 API 读取并同步店铺数据，您需要在开始迁移前先在 Shopify 中开发应用并获取 API 访问权限，具体步骤如下：

1. 登录 Shopify 后台，进入 **设置** -> **应用和销售渠道**。
2. 点击 **开发应用**，然后点击页面右上角的 **创建应用**。
3. 【首次设置】如为首次创建应用，系统将提示你启用自定义应用开发。请根据提示完成启用后，再次点击 **创建应用**。
4. 在 **应用名称** 处填写应用名称，选择 **应用开发者** （店主或有应用开发权限的员工）。点击 **创建应用**。
5. 在 **概览** 页签，点击 **配置 Admin API 访问范围**，勾选所有以 `read_` 开头的权限项，并点击 **保存**。
6. 返回 **概览** 页签，点击 **安装应用**。
7. 在确认弹窗中，点击 **安装** 继续操作。
8. 在 **API 凭据** 页签的 **Admin API 访问令牌** 部分，点击 **显示令牌一次**，获取 **后台 API 访问令牌**。**注意**：当前访问令牌仅可查看一次，请妥善保存。

::: tip 

在 Shopify 平台中，部分客户信息（如姓名、地址、电子邮箱和电话号码）被归类为 PII（Personally Identifiable Information，个人身份信息），属于受保护数据。  

只有订阅 **Shopify Grow**、**Shopify Advanced** 或 **Shopify Plus** 套餐的商家，其应用程序才能通过 API 访问这些敏感数据。

除 PII 外，平台上的其他数据（如运费、订单、礼品卡等）虽然访问权限相对宽松，但仍需经过授权后才能访问。

:::

### 步骤二：打开 Genstore 迁移助手

1. 登录 Genstore 商户后台，导航至 **应用**。
2. 在 **已安装** 页签，找到 **Migration Tool**。
3. 点击 **打开应用**，选择 **Shopify** 作为迁移来源，点击 **开始迁移**。

### 步骤三：店铺授权

1. 输入待迁移的 Shopify 店铺域名。
::: tip
**注意**：请填写以 `myshopify.com` 结尾的完整域名（例如：`sample.myshopify.com`）。不支持自有域名（如 `brand.com`）。
:::
2. 输入访问令牌，即 **后台 API 访问令牌**。
3. 完成设置后，点击 **下一步**。

### 步骤四：选择迁移模块

Genstore 支持将 Shopify 的以下模块迁移至 Genstore：

|模块|是否必选|依赖模块|
|---|---|---|
|店铺|✅ 必选|—|
|客户|可选|店铺|
|产品|可选|店铺|
|订单|可选|客户、产品|
|折扣|可选|客户、产品|
|自定义页面|可选|店铺|
|博客|可选|店铺|
|URL 重定向|可选|店铺|
|菜单导航|可选|店铺|
|主题模板|可选|店铺|

::: tip

- **迁移订单模块前，建议先迁移产品和客户。** 这样可确保订单在迁移时能够正确关联至 Genstore 中的客户和产品信息。
- **主题迁移支持 Shopify 官方 13 套主题**（如 Dawn、Spotlight、Sense、Craft、Ride 等），相关内容如产品、页面、博客、菜单将自动关联迁移。
:::

各模块迁移的常见问题，可阅读 [Shopify 店铺迁移常见问题解答](./start-migrate-from-shopify-faq.md)。

### 步骤五：自定义迁移设置

您可以根据筛选条件选择特定的数据进行迁移：

- 产品：按状态筛选（已发布、草稿、已归档等）。
- 订单：按订单创建时间筛选（如 2024-01-01 至 2024-12-31）。
- 折扣：是否迁移已禁用、已过期的折扣码。

### 步骤六：开始迁移

1. 确认所有设置后，点击 **开始迁移**。
2. 系统将开始处理迁移，您可在应用界面查看迁移进度。迁移时间取决于数据量大小，请耐心等待。
3. 迁移完成后，可进入 Genstore 后台进一步完善店铺设置，推荐阅读 [店铺快速上线 Checklist](./start-quick-launch-checklist.md)

## 查看迁移记录

完成店铺迁移后，您可在迁移助手应用内查看所有迁移历史记录，包括：

- **迁移时间**
- **迁移平台**（Shopify）
- **店铺域名**
- **迁移模块**
- **迁移数据结果**
- **查看异常详情（并支持导出异常数据，便于后续核对与排查）**

完成迁移后，请使用此清单确认您的商店设置是否齐全，并确保客户可以顺利访问和使用您的在线商店。

## 上线前设置

完成迁移后，请继续完成以下配置，确保您的商店功能完善、顺利上线：

### 步骤 1：完善店铺基础信息

路径：**设置 -> 通用**  
确保以下信息准确无误：

- 店铺名称与联系方式
- **默认货币**（确保与支付系统币种一致）
- **时区设置**（影响订单时间、报表分析等）
- 品牌 Logo、品牌色与社交媒体链接

### 步骤 2：配置域名并设置 URL 重定向（如适用）

路径：**设置 -> 域名**

- 绑定自定义域名或修改默认的 `.genmystore.com` 域名
- 设置原网址的 **URL 重定向**，维护 SEO 权重和老用户访问习惯

### 步骤 3：核对产品展示与库存设置

路径：**商店 -> 产品**

- 确认产品分类、标题、描述、图片、价格与库存无误
- 可根据需求重新归类到产品系列中

### 步骤 4：选择模板并完成店铺装修

路径：**商店** -> **在线商店** -> **模板**

- 选择主题模板，调整布局、颜色与内容
- 使用装修编辑器编辑首页、产品页、系列页等
- 可选择通过代码编辑器进行高级定制      

### 步骤 5：设置公共信息页（政策页）

- 创建隐私政策、退换货政策、服务条款等页面。路径：**设置 -> 政策**
- 导航设置路径：**商店** -> **在线商店** -> **菜单**
 - 页面启用路径：**商店** -> **在线商店** -> **模版**

### 步骤 6：配置物流与税费

- 物流：**设置 -> 发货配送**
    - 设置发货地址、仓库、配送区域与运费
    - （可选）绑定物流服务商，启用面单管理
- 税费：**设置 -> 税费**
    - 启用自动税率或手动设定适用税率

### 步骤 7：设置支付方式与结账流程

- 支付：**设置 -> 支付**
    - 启用 **Genstore Payments** 或第三方支付

- 结账流程：**设置 -> 结账页**
    - 设置所需的客户信息项（邮箱、电话、地址）
     - 配置支付授权模式（自动 / 手动）

### 步骤 8：预览店铺前台并检查所有页面与链接

- 检查首页、产品页、购物车页、结账页等页面展示是否正确
- 测试导航链接有效性，检查是否存在死链
- 预览移动端与桌面端响应式效果
- 确认各页面的 **SEO 元数据**（标题、描述、URL）

### 步骤 9：创建测试订单并验证关键功能

- 模拟正常下单、支付、发货流程
- 模拟失败交易、退款、取消订单等情况
- 验证通知邮件（如订单确认、发货通知）是否正确发送

### 步骤 10：关闭密码保护并正式对外开放商店

- 路径：**商店** -> **在线商店** -> **偏好设置**
- 移除访问密码，开放访客访问权限
### 步骤 11（可选）：发送客户账户激活邮件
 

 如迁移了客户数据，可通过批量发送密码重置链接，帮助客户重新登录账户。

### 步骤 12（可选）：设置 SEO、添加销售渠道与启动营销

- 配置 SEO 关键词与页面元信息
- 添加销售渠道（如 Google、Facebook）
- 启动营销计划，如邮件营销、社交媒体推广、优惠活动等
