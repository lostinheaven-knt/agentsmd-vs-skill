

# 迁移 Amazon 店铺

通过 Genstore 迁移助手，您可以将 Amazon 店铺的产品与订单数据一键迁移至 Genstore，实现多平台数据同步。请按照以下步骤完成迁移准备、授权及模块设置。

## 迁移准备

- 拥有有效的 Amazon 卖家账户，已开通目标区域市场。
  如尚未开通相应站点，请访问 [Amazon Seller Central](https://sellercentral.amazon.com/) 完成注册。
- Amazon 店铺所属国家或地区与 Genstore 店铺中已启用的市场一致。
- 如待迁移店铺所在市场尚未在 Genstore 中添加，请先前往 Genstore 商户后台 [新增对应市场](./global-market-manage-market.md) 后再进行迁移。
- [已安装 Genstore 迁移助手](./start-migrate-guide#安装-genstore-迁移助手)。

##### 远东及大洋洲

新加坡（`amazon.sg`）、澳大利亚（`amazon.com.au`）、日本（`amazon.co.jp`）

## 授权 Amazon 店铺

1. 打开 Genstore 迁移助手，点击选择 **Amazon 迁移至 Genstore**。
2. 点击 **开始迁移**。
3. 在 **选择市场** 下拉菜单中，选择目标 Amazon 站点，然后点击 **获取链接**。
4. 登录 Amazon Seller Central 并完成授权。
4. 授权完成后，系统将显示已授权的 Amazon 店铺信息，最多展示 10 个。

您可在 **已授权连接的市场** 页面查看：

- 店铺名称
- 所属站点域名（如 `amazon.com`）
- **立即迁移** 按钮（进入模块设置）
- 删除按钮（移除授权）

## 选择迁移模块

当前支持迁移以下模块：

| 模块           | 数据范围                                       |
| ---------------- | ------------------------------------------------ |
| **产品** | 名称、价格、描述、图片、规格、分类、状态等     |
| **订单** | 产品信息、金额（保留原币种）、收货人信息、状态等 |

:::tip

 勾选订单模块时，系统会自动勾选产品模块。

:::


## 产品模块设置

### 产品类型映射（必填）

操作说明：

* 勾选 **产品模块** 后，系统会自动展开 **Amazon 迁移产品类型映射** 设置区域。
* 为每个 Amazon 产品类型匹配 Genstore 类别。

::: tip

* 至少完成一组 **Amazon → Genstore** 产品类型映射
* 未完成映射的产品将不会被迁移

:::

### 自定义筛选条件

#### 产品筛选

* **产品状态**：可选择仅迁移 **可售** 产品，未筛选则默认迁移全部产品。
* **迁移地点**：仅支持迁移至 Genstore 中已启用的地点，默认使用系统设定地点。

#### 订单筛选

* **时间范围**​：按订单 **创建时间** 筛选，最大时间跨度为近一年。
* **订单状态**​：当前仅支持迁移  **已完成** 状态订单。

## 常见问题

### Q1. 未能授权 Amazon 店铺，怎么办？
> A：请确认您的 Amazon 卖家账户已开通目标市场，并在授权过程中同意访问权限请求。

### Q2. 为什么部分产品未迁移？
> A：请检查是否完成了产品类型映射。未完成映射的产品将被系统自动过滤。

### Q3. 可以只迁移部分产品吗？
> A：可通过 **产品状态** 筛选迁移范围。暂不支持按 SKU 或类目进一步筛选。

:::
