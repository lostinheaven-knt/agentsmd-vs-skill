# 管理余额

余额页面为商户提供账户资金的全面视图和管理工具，便于实时监控资金动态，并解决财务问题。本页面主要内容包括：

- 查看和跟踪账户余额。
- 解读余额仪表板中显示的关键指标和数据点。
- 有效处理调整并解决差异问题。

## 如何访问

登录 Genstore 商家后台，在左侧导航栏中，点击 **财务** -> **余额**。您可在余额页面查看账户中的资金总额和详细的结算历史。

::: tip

您需要首先 **开通 Genstore Payments** 才能正常使用 **财务** 功能。

:::

## 可用余额

页面顶部显示账户总额及其构成，其中，**可用余额** 是所有资金（结算中、已结算）总和，可用于退款和提现。

- **可用余额**：表示账户中正在清算和结算的资金，您可以使用这些资金来进行退款或者处理拒付。
  - 您可以通过可用余额下面的两个小数字分别查看自有产品交易可用余额（左侧数字）和 Dropshipping 产品交易的可用余额（右侧数字）
  - 注意：如果这些资金中包含 Dropshipping 订单的资金，那么退款的处理方式则需要符合 [Dropshipping 的退款约束](./operate-dropshipping-after-sales-policy.md)
- **清算中**：处于清算中的资金。
- **结算中**：清算成功处于结算状态的资金。
- **可提现金额**：结算成功。资金将根据您在 Genstore Payments 设定的付款计划自动拨付。
- **储备金**：为保证买家和卖家的权利，部分已结算的资金将会进入储备金账户，用于覆盖可能的退款、拒付等费用。储备金中的资金归属权仍属于商户。当订单完成后，储备金内的账户余额将会解冻。其中，**储备金阈值** 是储备金的最低额度要求，只有当保证金的余额高于阈值时，资金才会被解冻。

## 清算历史

页面下方展示所有处于清算阶段的支付单批次详细信息，帮助商家了解资金变动情况。

| 字段名称 | 说明                                                         |
| :------- | :----------------------------------------------------------- |
| 批次     | 清算批次的唯一编号                                           |
| 日期     | 清算计划完成的日期                                           |
| 状态     | 清算批次单的状态，一般情况下分为：`清算中`，`清算成功`           |
| 收入     | 支付单按照结算货币兑换后的待入账总金额                       |
| 退款     | 清算周期内产生的所有退款总额                                 |
| 费用     | 清算周期内产生的所有费用，包括支付处理费、外汇转换费或者 Dropshipping 相关的费用 |
| 调整额   | 由于其他原因造成的结算资金差异调整                           |
| 净额     | 收入扣除各项调整项后，剩余的实际进入可用余额的金额           |

## 结算历史

页面下方展示所有结算订单的详细信息，帮助商家了解资金变动情况。

| 字段名称     | 说明                                                         |
| ------------ | ------------------------------------------------------------ |
| 时间         | 资金变动产生的时间                                           |
| 动作         | 表示这次资金变化的动作类型，包括清算完成、产生拒付、发起转账、产生退款、余额调整、商户充值 |
| 清算参考号   | 如果这次资金变化与清算批次有联动，则可以在这里看到对应的参考号进行追踪 |
| 金额         | 本次资金变化的总金额                                         |
| 剩余可用资金 | 最终资金变化的结果                                           |
| 储备金       | 本次资金变化对于储备金的影响，向储备金添加资金展示为正数，从储备金提取资金展示为负数 |
| 负余额       | 本次资金变化对于负余额的影响，向负余额添加资金展示为正数，从负余额提取资金展示为负数 |
| 其他         | 本次资金变化对于其他类型资金的影响，目前当您使用了 Dropshipping，且因为需要退款而额外进行充值时，您可以通过这一列查看对应资金的变化 |

## 付款历史

您的资金处于可付款余额时，会按计划定时向您的银行账户付款，每次付款活动都会记录

| 字段名称   | 说明                               |
| ---------- | ---------------------------------- |
| 时间       | 发起付款的时间                     |
| 类型       | 动作类型，一般情况下是`付款`或者`转账` |
| 金额       | 本次付款的金额                     |
| 交易参考号 | 本次付款动作的唯一编号             |
| 描述       | 本次付款的额外说明，若有则会展示   |


## 筛选功能

结算历史页面右侧提供了强大的筛选功能，支持按日期和状态对结算记录进行快速过滤，帮助商家高效查找特定记录，

1. 点击筛选按钮打开筛选面板。
2. 根据需要选择日期范围或状态条件。
3. 点击 **OK** 应用筛选条件，或选择 **清除** 清除所有条件并显示完整记录。
