# Shopline 店铺迁移

本指南介绍如何使用 **Genstore 迁移助手** 将您的 Shopline 店铺相关数据（包括店铺信息、产品 ）迁移至 Genstore。

## 开始迁移

::: tip

- 店铺数据量较大时，迁移可能耗时较长。
- 迁移过程中请勿修改 API 权限，否则可能导致迁移失败。
- 尽量避免在迁移过程中对 Shopline 店铺进行大规模数据修改，以防数据不一致。
- [安装 Genstore 迁移助手](./start-migrate-guide#安装-genstore-迁移助手)。
:::

### 步骤一：获取 Shopline API 访问权限

由于迁移过程中需要通过 API 读取并同步店铺数据，您需要在开始迁移前先在 Shopline 中开发应用并获取 API 访问权限，具体步骤如下：

1. 登录 Shopline 后台，进入 **应用**，点击页面右上角的 **开发应用** 启动应用开发流程。
2. 点击 **创建应用**，填写应用名称并选择 **应用开发者**，系统会自动填写 **应用联系人邮箱**，完成后点击 **创建应用**。
3. 在返回的应用列表页中，点击新建应用 **操作** 栏的 **编辑** 进入权限配置。
4. 在 **集成后台 API** 一栏点击 **配置**，勾选所有以 `查询` 开头的权限项（如 `查询商品`），完成后点击 **保存**。
5. 可在 **权限配置** 页签查看已授权的所有权限项。
6. 在 **API 凭据** 页面点击 **安装应用**，获取后台 API 访问令牌。
7. 在 **API 凭据** 页签中，找到 **使用您的访问令牌从后台 API 请求数据** 部分，点击显示图标查看访问令牌。
8. 为保障数据安全，系统将提示身份验证。请按提示完成并妥善保存访问令牌。**注意**：访问令牌仅可查看一次，请务必保存好。

### 步骤二：打开 Genstore 迁移助手

1. 登录 Genstore 商户后台，导航至 ​**应用**​。
2. 在 **已安装** 页签，找到 ​**Migration Tool**​。
3. 点击 ​**打开应用**​，选择 **Shopline** 作为迁移来源，点击 ​**开始搬迁**​。

### 步骤三：店铺授权

1. 输入待迁移的 Shopline 店铺域名。
	::: tip
	​请填写以 `myshopline.com` 结尾的完整域名（例如：`sample.myshopline.com`）。不支持自有域名（如 `brand.com`）。​		
	:::
2. 输入访问令牌，即 ​**后台 API 访问令牌**​。
3. 完成设置后，点击 ​**下一步**​。

### 步骤四：选择迁移模块

Genstore 支持将 Shopline 的以下模块迁移至 Genstore：

| 模块       | 是否必选
| ------------ | ---------- |
| 店铺       | 可选  |
| 产品       | 可选   |

::: tip

在迁移产品时，您可以选择 **覆盖更新** 方式： 系统将根据已有的产品数据进行匹配，对已重复产品内容进行数据跳过处理，而非创建重复项。

:::


### 步骤五：自定义迁移设置

您可以根据筛选条件选择特定的数据进行迁移：

* 产品：按状态筛选（上架、下架、归档）。

### 步骤六：开始迁移

1. 确认所有设置后，点击 ​**开始迁移**​。
2. 系统将开始处理迁移，您可在应用界面查看迁移进度。迁移时间取决于数据量大小，请耐心等待。
3. 迁移完成后，可进入 Genstore 后台进一步完善店铺设置，推荐阅读 [店铺快速上线 Checklist](./start-quick-launch-checklist.md)

## 查看迁移记录

完成店铺迁移后，您可在迁移助手应用内查看所有迁移历史记录，包括：

* **迁移时间**
* **迁移平台**​（Shopline）
* **店铺域名**
* **迁移模块**
* **迁移数据结果**
* **查看异常详情（并支持导出异常数据，便于后续核对与排查）**
