# Balance 页面说明

通过 Drop Pay 收取的资金，统一在 **Dropshipping** -> **Balance** -> **Drop Pay** 视图下进行管理。您可以在此实时监控资金的流转状态，从支付、清算、结算到最终转账。

## 核心指标
- **可用余额**：显示当前可用于退款或处理纠纷的总金额，包含总收入、清算中与结算中的金额。注意：只有结算成功的金额才有可能向您付款。
- **储备金**：为保证买家和卖家的权利，部分已结算的资金将会进入储备金账户，用于覆盖可能的退款、拒付等费用。储备金中的资金归属权仍属于商户。当订单完成后，储备金内的账户余额将会解冻。其中，**储备金阈值** 是储备金的最低额度要求，只有当保证金的余额高于阈值时，资金才会被解冻。
- **应付余额**：已完成结算并准备好向您的 PayPal 账户付款的金额。

## 交易全流程监控
您可以通过页面下面的列表区域查看所有通过 Drop Pay 交易的支付单

### 支付明细
展示全部交易的支付单状态（已授权、已退款等）、风险评分及每笔单据扣除成本后的净额。

| 列表名称   | 说明                                                         |
| ---------- | ------------------------------------------------------------ |
| 创建时间   | 这笔交易的支付单创建的时间                                   |
| 支付参考号 | 这笔支付单的唯一编号，点击支付参考号，可以从页面侧边滑出支付单详情，您可以查看支付的具体详情，包含了支付信息、处理费、Dropshipping 各项成本扣除展示、退款情况，您可以通过底部的时间轴查看事件处理的时间和详情 |
| 状态       | 支付单的状态，一般情况下会包含：已授权、已捕获、已取消、已退款、部分退款等等 |
| 订单       | 这笔支付单关联的订单编号，一般情况下一笔支付单仅会对应一个订单编号，如果消费者的某个结账行为中同时存在实物产品和数字产品，则订单会出现多个，那么这里就会展示多个订单。点击订单编号则会打开订单详情页|
| 付款人     | 购买产品并产生支付行为的客户邮箱                   |
| 支付方式   | 客户在结账时使用的支付方式，比如信用卡、Apple Pay 或者 Klarna 等等。 |
| 总金额     | 客户支付的原始总金额，以支付时使用的货币标识             | 
| 净额       | 这笔支付单在扣除部分费用后剩余的最终金额，并且以 Drop Pay 结算的货币标识 |
| 风险评分   | 您可以通过风险评分评估风险数据，风险越高则评分越大，点击评分可以展示一些风险元素分析，包含账单地址检查结果、配送地址与 IP 地址检查结果、下单 IP、是否使用 3D-Secure、CVC 验证结果等等。 |

## 清算历史

本列表展示了通过 Drop Pay 处理的支付清算批次信息，一般情况下，同一个自然日的支付单将会出现在同一个清算批次中，完成清算将会进入结算，您可以通过这个列表来跟踪您的资金在途情况。关于清结算周期的详细说明，可参考 [资金分账与结算规则](./operate-dropshipping-fund-splitting.md)

| 字段名称 | 说明                                           |
| :------- | :--------------------------------------------- |
| 批次     | 清算的唯一编号                                 |
| 日期     | 清算批次预计完成的时间                         |
| 状态     | 清算批次单的状态，包含 `清算中` 与 `已清算` 两种状态 |
| 收入     | 当前清算批次中总交易金额，以 `USD` 展示            |
| 退款     | 当前清算批次中产生的退款总额                   |
| 费用     | 处理支付交易产生的手续费费用                   |
| 调整额   | 由于其他原因造成的清算资金的差异调整             |
| 净额     | 收入扣除各项调整项后，剩余的实际可以结算的金额 |

## 转账付款

记录系统向您的 PayPal 账户发起付款的详情。请注意，实际到账金额会扣除一定的付款手续费。

| 字段名称   | 说明                                      |
| ---------- | ----------------------------------------- |
| 开始日期       | 向您的 PayPal 账户发起付款的时间            |
| 类型       | 付款的类型展示                            |
| 收款账户   | 您的 PayPal 账户邮箱                      |
| 付款金额   | 本次计划向您付款的总金额，以 `USD` 展示       |
| 费用       | 本次计划付款中收取的付款手续费，以 `USD` 展示 |
| 转账参考 | 本次付款的唯一参考号                      |
| 描述       | 付款描述                                  |

::: tip

* 资金的实际到账时间以 PayPal 的处理周期为准，不同国家/地区的到账时间可能存在差异
* 请确保您的 PayPal 账户状态正常且信息真实完整，否则可能影响资金到账。
* 实际到账的金额与付款金额会有差异，其中会产生付款手续费，您可以在付款记录中查看收取的付款手续费的费用

:::

## 处理纠纷

如果您售卖 Dropshipping 产品时，您的客户向持卡机构发起纠纷或者拒付，将会在这里出现，您可以在这里处理纠纷。一般情况下，您可以提交您的履约证明、您与客户沟通的记录或者其他更多证明材料进行抗辩。更多内容，可参考 [拒付和查询](./operate-genstore-payments-chargebacks-overview.md)
