# 产品定价设置

在 **Genstore Dropshipping** 中，您可以通过设置批量定价规则，实现产品价格的自动计算与更新，减少人工维护工作量。
系统会根据您设定的加价逻辑（固定金额或比例加价）计算产品价格与原价，并可选择是否将运费计入定价。

## 功能价值

* 批量计算并更新产品价格，减少人工操作
* 灵活选择固定或比例加价，满足不同利润策略
* 可按区间区分定价，实现多层级利润控制

## 定价规则

系统支持同时设置 **产品定价** 与 **区间定价规则：**

- **产品定价**：适用于所有产品的统一加价逻辑。
- **区间定价规则**：适用于特定价格区间或条件下的加价逻辑。

当两者同时存在时，区间定价规则优先级更高。
系统会优先根据区间规则计算对应价格，未命中的部分才会使用产品定价规则。

## 产品定价

在 **产品定价** 区域，您可以为所有代发货的产品分别设置 **产品价格** 和 **产品原价**（划线价） 。
系统将在产品推送至店铺时，根据成本价自动计算售价与原价。

#### 规则类型

| 定价方式       | 说明                            |
| ---------- | ----------------------------- |
| 成本价/原价 + 固定金额 | 适用于所有产品统一加价金额的场景（如 成本 + 3 元）。 |
| 成本价/原价 × 比例   | 适用于按利润率自动计算产品价格的场景（如 成本 × 150%）。 |
| 是否包含运费     | 若选择 **包含运费**，系统将在计算产品价格时自动将运费计入成本。 |

#### 操作步骤

1. 进入 **Dropshipping 设置 → 产品定价**。
2. 选择定价模式（成本价/原价 + 固定金额 / 成本价/原价 × 比例）。
3. 在下拉框中选择 **包含运费** 或 **不含运费**。
4. 点击 **保存**，系统将自动计算产品价格和原价。

> **提示：**
> 修改定价规则后，仅对新推送的产品生效，已推送的产品不会自动更新。

## 区间定价规则

系统支持分别为 **产品价格** 和 **产品原价**（划线价） 设置独立的定价区间，从而在实际销售与展示价格上实现更灵活的控制。

#### 功能说明

- **产品价格**：用于买家实际支付的价格计算。
- **产品原价**：用于前台展示的划线价（通常用于显示折扣前价格）。
两者可分别设置不同的区间和加价方式，系统将自动根据成本价应用相应计算。

#### 规则类型

| 定价方式       | 说明                            |
| ---------- | ----------------------------- |
| 按价格区间设置固定加价 | 适用于按固定金额加价的场景（如 成本 + 3 元）。 |
| 按价格区间设置比例加价   | 适用于按利润率自动计算产品价格的场景（如 成本 × 150%）。 |
> 均可选择是否包含运费

#### 操作步骤

1. 进入 Dropshipping 设置 → 区间定价。
2. 在 **产品价格** 或 **产品原价** 模块中点击 **添加规则**。
3. 输入区间范围（例如 ￥0–￥100 / ￥100–￥300）。
4. 设置加价金额或比例，并选择是否包含运费。
5. 点击 **保存**，系统将在产品推送时自动根据成本价分别计算产品价格与原价。

示例

| 成本区间     | 产品价格加价比例 | 原价加价比例 |
| -------- | ------ | ------ |
| ￥0–￥50   | 50%    | 80%    |
| ￥50–￥100 | 30%    | 50%    |


## 价格显示优化

为让产品价格更符合消费者的心理预期（如 ￥9.99、￥19.95），您可以启用 **价格显示优化** 功能。
系统会在计算结果基础上自动调整价格尾数，使产品价格更美观统一。

### 操作步骤

1. 勾选 **设置价格小数部分** 或 **设置原价小数部分**。
2. 输入希望显示的价格尾数（例如 .99 或 .95）。
3. 点击 **保存**，系统将在计算价格时自动应用相应的尾数规则。

## 主要发送到的国家

**主要发送到的国家** 用于确定定价时的运费计算基准。
系统会以该国家为基础，自动匹配产品运费数据，用于计算产品价格和利润。

> 例如：若设置为“美国”，系统会自动获取发往美国的运费作为定价计算依据。
