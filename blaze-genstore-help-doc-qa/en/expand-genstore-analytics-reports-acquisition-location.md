# Sessions by location

The **Sessions by location** report provides granular **geographic insights** into your website traffic, including visitor counts, sessions, and page views segmented by **city, region, and country/region**. This report helps you identify high-potential markets and assess regional user engagement.

## Use this report to

- **Global traffic overview**: A heatmap displays session distribution across countries/regions, enabling quick identification of high-activity markets.
- **Geographic drill-down**: Click any country/region on the heatmap to **drill into regional and city-level data**, uncovering localized trends and opportunities.
- **Location performance**: Use the detailed location list to compare metrics (e.g., bounce rates, session duration) across cities or regions, informing market prioritization.

## Report data fields

For general terms, refer to [Acquisition reports - common terms](./expand-genstore-analytics-reports-acquisition.md#common-terms).

This report includes the following additional terms:

| Term | Definition |
| ---- | ---------- |
| **City** | The specific city (e.g., Beijing, New York) where a user accessed your website. Used to analyze city - level engagement and prioritize local campaigns. |
| **Region** | The broader administrative area (e.g., Guangdong Province, California) where visits originated. Helps evaluate regional market potential. |
| **Country/region** | The country or territory (e.g., China, United States) of the visitor. Guides global expansion and localization strategies. |

:::tip

Geographic data accuracy may vary due to user privacy settings (e.g., VPNs) and IP address resolution limitations. Treat this data as directional rather than absolute.

:::
