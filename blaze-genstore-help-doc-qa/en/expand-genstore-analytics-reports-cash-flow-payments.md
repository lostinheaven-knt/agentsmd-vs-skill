# Payments

The **Payments report** provides insights into **payment behavior patterns**, such as peak transaction times (e.g., midday and evening), helping identify cyclical trends. It also details payment methods used by customers, accelerated checkout services (e.g., PayPal, Apple Pay), credit card brands/types, and billing countries/regions, enabling you to analyze payment preferences and market potential.

## Report data fields

| Term | Definition |
| ---- | ---------- |
| **Transaction timestamp** | The exact date and time when the payment was successfully processed. |
| **Payment method** | The method used by the customer (e.g., credit card, bank transfer). If marked as "paid" with an alternative method, this displays as "Manual." |
| **Accelerated checkout service used** | The accelerated checkout service (if applicable), such as PayPal or Apple Pay. |
| **Credit card** | The brand of the credit card used for the transaction (if applicable). |
| **Credit card type** | The type of credit card (e.g., Standard, Premium). |
| **Billing country/region** | The country/region associated with the order’s billing address. |
| **Sales channel** | The platform or channel where the order or return was placed. |
| **Transactions** | The total number of payment transactions. |
| **Payment** | The total amount paid before refunds. |
| **Refund** | The total refunded amount across all transactions. |
| **Net payment** | Calculated as `Total payment - Refund`. |
