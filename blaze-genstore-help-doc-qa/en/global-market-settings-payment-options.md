# Configure payment methods by market

When you create a new market, the system will automatically load all payment methods already enabled in your store. When you add a new payment method, it will by default be applied to all existing markets.

When selling in multiple markets, you can display different payment methods to customers in different countries/regions, instead of showing all payment methods across all markets.

::: tip

- **Genstore Payments**: The system automatically displays payment methods based on the customer’s country/region or currency, with no extra configuration required.
- **Other third-party payment methods**: You can configure which payment methods to display for each market.  
:::
## Steps to configure

1. In the **Genstore admin**, go to **Settings -> Markets**.
2. In the **Overview** page, select the market you want to configure.
3. Enter the market page and click **Payments**.
4. In the payment options list, check or uncheck the methods you want to display for that market.
5. Click **Save**.

## Example on storefront

Suppose you have enabled multiple local payment options under a third-party payment provider:

- For the **Japan market**, select only **PayPay**.
- For the **Thailand market**, select only **TrueMoney**.

This way, customers in Japan will only see PayPay at checkout, while customers in Thailand will only see TrueMoney, ensuring a payment experience tailored to local preferences.
